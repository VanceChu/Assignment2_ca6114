## Report A (Van Gogh) — 思路讲解

### 叙事主线
**"LoRA 能把梵高的笔触和色彩迁移到 SDXL 输出上，但 prompt 越复杂，风格越不稳定。"**

这是一份**传统对比式**报告，按线性逻辑推进：我做了什么 → 怎么评估 → 发现了什么 → 哪里不够好。

### Section 1: Introduction（~350 词）
做了两件事：
1. **介绍 LoRA 是什么**——低秩分解、参数高效、可即插即用。写给不了解 LoRA 的老师看
2. **定义研究问题**——"LoRA 能否迁移梵高 1888 阿尔勒时期的笔触纹理？prompt 复杂度升高时风格会怎样衰减？"

选 1888 的理由：这是梵高风格最集中、最一致的时期（厚涂笔触、暖色调、方向性标记），为 LoRA 提供干净的训练信号。

### Section 2: Dataset and Training（~800 词）
分三个子节：

**2.1 数据集筛选**：40 张候选 → 30 张入选。三条筛选标准清楚写出来：时间一致性（全是 1888）、分辨率 ≥1024、题材多样性（肖像/风景/静物/室内/城市）。

**2.2 Caption 策略**：展示 3 条代表性 caption，解释 trigger token 和固定后缀的设计。**关键埋伏**：这里明确说"统一模板有局限"，为 Section 5 的局限讨论做铺垫。

**2.3 训练配置 + ComfyUI**：先给完整超参数表，然后解释每个关键参数的选择理由（rank 16 为什么合适、双学习率为什么这么设、cosine 调度器的好处）。**ComfyUI 部分**从这里展开——描述 8 节点 workflow 的结构、每个节点的功能、最终选定的推理参数（Euler/30 steps/CFG 7）。

### Section 3: Evaluation Design（~350 词）
定义 72 图矩阵（6 prompt × 3 seed × 4 weight），解释三档 prompt 的设计逻辑。**核心假设**在这里提出：摄影语义（rim light, 85mm lens, volumetric light）会和 LoRA 的绘画风格产生冲突。声明正文只详细分析 3 个 prompt（portrait / bookstore / museum hall），其余放附录。

### Section 4: Findings（~1500 词）——报告核心
这是最厚的一节，结构是"评价表 + 三段逐 prompt 深度分析 + 跨层级总结"。

**4.1 评价表**：一张总结表先给老师全局印象——simple 风格高/美感高/写实低，medium 中等，complex 风格中/写实低。

**4.2 Simple（portrait, seed 202）**：最成功的一组。详细描述 baseline 是什么样 → w=1.0 变成了什么样 → 为什么成功（prompt 词汇和梵高自己的题材兼容，没有摄影语义干扰）。还提到 w=0.6 → 0.8 → 1.0 的渐变效果。

**4.3 Medium（bookstore, seed 202）**：中间地带。重点发现："warm ambient lighting"这种描述意境的词和梵高暖色调天然兼容，所以风格迁移效果好。空间结构保留完好——梵高自己的画也保持清晰空间组织。得出结论：**prompt 的词汇比场景复杂度更重要**。

**4.4 Complex（museum hall, seed 101）**：最极端的变化。天花板变成星空式漩涡、地板变成黄绿色条纹、人物变成剪影。分析这是"语义竞争"——prompt 要 volumetric light（摄影域），LoRA 要 painterly texture（绘画域），w=1.0 时绘画赢了，但内容被重新解读而非精确还原。

**4.5 跨层级总结**：提炼 4 个 pattern——色彩偏移最稳健、笔触随 weight 线性增强、空间容忍度随复杂度下降、摄影词汇是风格衰减的最强预测因子。

### Section 5: Limitations + Failure（~700 词）
4 个局限 + 2 个 failure case：

1. **Caption 模板统一**——模型可能学到"expressive brushstrokes"这个词而非实际视觉结构（天空的长弧线 vs 树叶的短点触 vs 面部的平滑过渡）
2. **没做 checkpoint 对比**——只用了 final，中间 step 可能更适合复杂场景
3. **无局部编辑能力**——全图统一风格化，无法区域控制
4. **ComfyUI 缺乏集成训练**——训练在 sd-scripts，推理在 ComfyUI，中间需要手动搬运 checkpoint，反馈循环太长

Failure cases：museum seed 202（内容完全崩溃）和 portrait seed 101（面部过度风格化）。归纳共同模式：**LoRA weight 应该根据内容复杂度调整，而非固定为 1.0**。

### Section 6: Conclusion（~300 词）
三句话版结论：
- 梵高 LoRA 适合"画意化"而非"高保真写实"
- 最佳应用场景是中低复杂度的肖像/室内/风景
- prompt 设计和 caption 策略对结果质量的影响不亚于模型架构

## Report B (Hokusai) — 思路讲解

### 叙事主线
**"LoRA 能让 SDXL 学到版画的结构特征（轮廓线、平涂、靛蓝色板），但摄影语义的 prompt 会和版画美学发生根本性冲突——而且冲突的结果不是'风格差'，而是'两种视觉范式在打架'。"**

这是一份**问题导向式**报告——不以"我做了什么"开头，而以"一个问题"开头，每一节都在回答这个问题的不同侧面。

### Section 1: Research Goal（~350 词）
**以问句开篇**："24 张江户时代版画能教会一个文生图模型画轮廓线、平涂色块和受限色板吗？"

然后解释为什么这个问题有意义：浮世绘版画的视觉特征（轮廓线、平面化色块、有限色板、非西方透视）与 SDXL 的训练分布（连续渐变、景深模糊、物理光学透视）**结构上不兼容**。这不是"换个滤镜"的问题，而是两种完全不同的视觉体系之间的冲突。

### Section 2: Dataset + Caption（~700 词）
分两个子节，**第二个子节是这份报告的方法论亮点**：

**2.1 选图**：24 张北斋风景画，解释三条约束（只选风景、只选北斋、不混其他浮世绘画家）。关键论点：约束不是限制，而是设计——把"artist"和"genre"固定，唯一变量是"scene content"，这样 LoRA 学到的是共性风格而非特定构图。

**2.2 Caption 设计作为方法贡献**：这是和 Report A 的核心差异。展示 3 条 caption，解释"固定风格骨架 + 变化场景描述"的二元结构。论证为什么这比"统一模板"更好——它在训练信号中**显式分离了 style 和 content**，减少了内容过拟合。还专门用一段反事实推理："如果用统一模板会怎样？→ 模型会记住波浪/富士山而非抽象风格特征。"

### Section 3: Training + Loss Curve（~800 词）
**这一节是 Report B 的独家内容**——Report A 没有 loss 曲线。

**3.1 配置表 + ComfyUI**：配置表与 A 结构相似但角度不同——侧重讲 ComfyUI 的 LoraLoader 双通道权重控制（strength_model vs strength_clip）和 KSampler 参数调节体验。

**3.2 Loss 曲线分析**：嵌入了 loss_curve.png，将 1200 步分成三阶段：
- Phase 1（0-400）：快速下降，学习粗粒度风格特征
- Phase 2（400-800）：波动平台期，不是过拟合，而是模型在"记忆单图"和"泛化风格"之间摇摆
- Phase 3（800-1200）：从 step 800 高峰回落，最终稳定在 0.123

**3.3 Checkpoint 选择**：承认 step 520 才是真正最低点（0.117），但选了 final（0.123）用于验证——差距只有 0.006，且 final 经历了 5 个完整 epoch。

### Section 4: Results（~1300 词）——报告核心
评价维度和 Report A 完全不同——不是"style fidelity / aesthetic / realism"，而是三个**结构性指标**：轮廓清晰度、色彩平面度、构图层次感。

**4.1 Simple（coffee cup, seed 303）**：最成功。咖啡杯变成了浮世绘器物——杯面出现波浪纹、碟子变成版画装饰、背景平涂化。关键发现："clean composition"这个 prompt 指令与浮世绘的留白特征天然兼容。

**4.2 Medium（fashion editorial, seed 101）**：**最有讨论价值的一组。** 这里发生了一件有趣的事：即使 LoRA weight=0（baseline），trigger token `<hokusai_style>` 已经让人物穿上了和服——说明 trigger token 不只是 LoRA 的激活信号，还是一个**语义内容修改器**。但到了 w=1.0，虽然和服更精致了，**渲染结构依然是摄影式的**——平滑皮肤渐变、写实布料褶皱、rim light 边缘高光。结论：prompt 里的"dramatic rim light"、"85mm lens"把模型锚定在写实渲染模式，LoRA 的 rank-16 改动不够强到翻盘。LoRA 成功改了**内容**（人物穿和服）但没改**结构**（依然是照片）。

**4.3 Complex（neon street, seed 202）**：**最意外的发现。** 本来预期这组会最差（霓虹夜市和 19 世纪版画毫无关系），但实际效果非常好！原因是 prompt 里的"layered depth"意外和浮世绘的**叠面式构图**（foreground/middleground/background 分离）对齐了。商铺立面平面化，人群图形化，空间层次反而比 medium 更接近版画。但色板仍然太现代——霓虹色溢出，靛蓝色板没有主导。

**4.4 重新定义评价标准**：这是一个元分析段落——LoRA 不是在"提升真实感"，而是在"降低摄影感、提升图形感"。用摄影保真度指标来评价它是错误的框架。

### Section 5: Discussion（~500 词）
上升到更高层面：这是**两种视觉范式的冲突**——西方摄影传统 vs 东亚版画传统。LoRA 的价值不是"让东西看起来更真"，而是"让东西看起来更像设计/图形"。

核心 insight：**语义距离不等于风格迁移难度。** 霓虹夜市（语义上离北斋很远）比时尚人像（语义上更近）反而效果更好——因为关键不是场景内容是否相似，而是 prompt 词汇是否与目标风格的视觉逻辑兼容。"layered depth"兼容浮世绘，"85mm lens"不兼容。

实践建议：不要只在 prompt 前面加 trigger token，要**用目标风格自身的构图语言重写 prompt**。

### Section 6: Future Work（~600 词）
6 个方向，每个都针对实验中观察到的具体局限：
1. **ControlNet 轮廓引导**——用 edge detection 强制保证轮廓线（针对 fashion 的失败）
2. **Style/Content 解耦**——分别训练构图 LoRA 和色彩 LoRA
3. **色板约束**——后处理或条件生成，把霓虹色映射回传统浮世绘色板（针对 neon street 的色彩溢出）
4. **Image-to-image**——用线稿/边缘图作为输入，保证结构完整性
5. **定量评估指标**——开发"版画感"指标（边缘密度、色彩直方图平坦度、梯度分布）
6. **ComfyUI 工具改进**——缺少原生 side-by-side 对比视图、缺少内置评估节点
